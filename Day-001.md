# ✅ Topics Covered

## 1. Database Indexing

### What I Learned

-   An index helps MySQL find data faster.
-   It works like the index of a book.
-   More indexes improve read performance.
-   Too many indexes slow down INSERT and UPDATE operations because
    indexes must also be updated.

### Composite Index

Example:

``` text
(city, treatment, hospital_name)
```

A composite index should match the application's most common search
pattern.

------------------------------------------------------------------------

## 2. Leftmost Prefix Rule

For an index:

``` text
(city, experience, rating)
```

These queries can efficiently use the index:

``` sql
WHERE city = 'Delhi'
```

``` sql
WHERE city = 'Delhi'
AND experience = 10
```

This query usually **cannot** efficiently use the index:

``` sql
WHERE rating > 4.5
```

because the search doesn't start from the leftmost indexed column.

------------------------------------------------------------------------

## 3. Equality vs Range Conditions

### Equality (`=`)

``` sql
WHERE city = 'Delhi'
AND experience = 10
```

MySQL jumps directly to the exact location in the index.

### Range (`>`, `<`, `BETWEEN`)

``` sql
WHERE city = 'Delhi'
AND experience > 10
ORDER BY rating DESC
```

The index is still used, but MySQL must scan a range of values. Because
of this, it often cannot use the remaining index columns as efficiently
for sorting.

------------------------------------------------------------------------

## 4. EXPLAIN

`EXPLAIN` shows how MySQL plans to execute a query.

Things to look for:

-   `type = ALL` → Full table scan (usually bad for large tables)
-   `key = NULL` → No index used
-   `rows` → Estimated number of rows MySQL will examine

------------------------------------------------------------------------

## 5. Choosing the Right Index

Instead of indexing every column, create indexes based on how users
actually search.

Example:

``` sql
WHERE hospital_id = 15
AND status = 'confirmed'
ORDER BY appointment_date DESC;
```

A good composite index would follow the query pattern.

------------------------------------------------------------------------

## 6. Lookup Table

A lookup table stores frequently searched data in a single optimized
table.

Instead of:

-   Multiple `wp_postmeta` rows
-   Multiple JOINs

Use one table like:

  hospital_id   city   rating   doctor_count   available_beds
  ------------- ------ -------- -------------- ----------------

Benefits: - Faster filtering - Simpler queries - Better scalability

------------------------------------------------------------------------

## 7. Why WooCommerce Uses Lookup Tables

WooCommerce stores searchable product information in:

``` text
wc_product_meta_lookup
```

instead of reading multiple `wp_postmeta` rows every time.

Orders are stored in dedicated tables (`wc_orders`) using HPOS for
better performance.

------------------------------------------------------------------------

## 8. Elasticsearch (Concept)

Elasticsearch is **not a replacement for MySQL**.

Think of it as a dedicated search engine.

Flow:

``` text
WordPress
    ↓
Save Doctor
    ↓
Send JSON
    ↓
Elasticsearch
```

Search Flow:

``` text
User Search
    ↓
Elasticsearch
    ↓
Doctor IDs
    ↓
WordPress Displays Results
```

It stores searchable documents in an optimized format, making complex
searches much faster than querying MySQL directly.

------------------------------------------------------------------------

# 💡 Biggest Takeaways

-   Always use the highest-level API that solves the problem.
-   Optimize reads by precomputing data when appropriate.
-   Use Custom Tables for transactional data.
-   Use CPTs for content.
-   Design indexes based on query patterns.
-   Measure before optimizing.
-   Understand the data model before writing code.
